1. Vue d'ensemble

Easy Medical est une application web complète de gestion pour établissements de santé. Elle centralise l'accueil patient, les consultations, les examens, la pharmacie, la facturation, la caisse et la comptabilité dans une interface unique et moderne.

Type d'application

Application web monolithique full-stack avec interface responsive et API internes.

Utilisateurs cibles

Médecins, infirmiers, réceptionnistes, caissiers, comptables, biologistes, pharmaciens.

Base de données

MongoDB via Mongoose, adaptée aux données médicales complexes et évolutives.

Technologies clés

Next.js 16, React 19, TypeScript 5, Bootstrap 5, PDFKit, Puppeteer, JWT.

3. Architecture générale

L'application repose sur une architecture monolithique Next.js répartie en couches claires :

  • Interface utilisateur : pages et composants React organisés par service métier.
  • API interne : routes REST Next.js couvrant l'ensemble des domaines fonctionnels.
  • Modèles de données : ~63 schémas Mongoose pour patients, actes, factures, stocks, etc.
  • Services utilitaires : impression, calculs, génération de documents, authentification.

5. Modèle de données

La base de données couvre les domaines métiers suivants :

  • Cœur : patients, utilisateurs, médecins, infirmiers, établissements.
  • Soins : consultations, examens, prescriptions, hospitalisations, rendez-vous.
  • Facturation : factures, lignes de facture, encaissements, annulations, honoraires.
  • Pharmacie & stock : produits, entrées/sorties, inventaires, fournisseurs.
  • Laboratoire : prestations, paramètres biologiques, résultats, automates.
  • Assurances : assurances, sociétés, tarifs conventionnés, partenaires.

7. Déploiement & prérequis

Prérequis techniques

  • Serveur supportant Node.js (compatible Next.js 16 / React 19).
  • Base de données MongoDB dédiée.
  • Variables d'environnement : MONGODB_URI, JWT_SECRET.
  • Navigateur web moderne pour les utilisateurs finaux.

Commandes principales

CommandeUsage
npm installInstallation des dépendances
npm run devDémarrage en développement local
npm run buildConstruction de l'application
npm run startLancement en production

Recommandations de déploiement

  • Hébergement sur serveur dédié, VPS ou PaaS compatible Node.js.
  • Activation des sauvegardes automatiques MongoDB.
  • Configuration d'un reverse-proxy HTTPS (Nginx, Apache, ou service cloud).
  • Vérification des dépendances système pour Puppeteer si génération PDF activée.

8. Points de vigilance

Note importante : l'application est en version initiale (0.1.0). Avant toute mise en production, il est recommandé de réaliser une phase de recette, de sécuriser les variables d'environnement, de mettre en place des sauvegardes et de formaliser les rôles/utilisateurs selon le contexte de l'établissement.
  • Créer un fichier .env.local avec les paramètres sensibles.
  • Harmoniser la gestion des erreurs API côté client.
  • Mettre en place un système d'audit des actions sensibles.
  • Prévoir des tests fonctionnels avant déploiement réel.
